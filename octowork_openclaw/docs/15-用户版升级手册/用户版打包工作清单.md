## 阶段一：代码改造（在开发版中完成） -- 已完成 2026-04-05

### 1.1 License 校验模块

- [x] 新建 `backend/src/utils/fingerprint.js` — 机器指纹生成 (PR #16)
  - `generateFingerprint()` 返回 32 位十六进制字符串
  - 六维度: hostname + platform + arch + cpuModel + totalMem + macAddress
- [x] 新建 `backend/src/license/verifier.js` — License 校验器 (PR #16)
  - `verifyLicense()` 启动时调用，校验不通过 → `process.exit(1)`
  - 校验签名（HMAC-SHA256）→ 校验机器指纹 → 校验过期时间
  - 未激活时打印机器指纹，引导用户获取授权
- [x] 新建 `backend/src/license/constants.js` — License 常量 (PR #16)
  - `LICENSE_SECRET` 密钥（已替换为随机 128 位十六进制，打包后编译进字节码不可见）
  - `LICENSE_PATH` 路径（`../../../license.key`，相对于 license/ 目录）
- [x] 修改 `backend/server.js` — 顶部集成校验 (PR #16)
  - 在 dotenv.config() 之后、Express 初始化之前加入 `verifyLicense()`
  - 添加 `/api/system/license-status` API（返回客户名/过期时间，不返回密钥）

### 1.2 版本检查模块

- [x] 新建 `backend/src/updater/checker.js` — 检查更新逻辑 (PR #16)
  - `checkForUpdate()` 从 GitHub manifest.json 获取最新版本
  - `getCurrentVersion()` 读取 `VERSION` 文件
  - `compareVersions(a, b)` 语义化版本比较
- [x] 修改 `backend/server.js` — 注册更新 API (PR #16)
  - `GET /api/system/check-update` — 前端查询更新
  - `GET /api/system/version` — 返回当前版本 + 平台信息
  - 启动后 10s 延迟执行首次检查（不阻塞启动）
- [x] 创建 `VERSION` 文件 — 项目根目录，内容 `1.0.0` (PR #16)

### 1.3 后端托管前端静态文件

- [x] 修改 `backend/server.js` — 添加 Express 静态文件托管 (PR #16)
  - `express.static('../frontend/dist')` 
  - SPA fallback: 非 `/api`、`/ws`、`/uploads` 路径返回 `index.html`
  - 仅当 `frontend/dist` 目录存在时启用（开发模式不影响）
- [x] 修改 `backend/server.js` — avatars 路径兼容 (PR #16)
  - 开发模式: `../frontend/public/avatars`
  - 生产模式: `../frontend/dist/avatars`
  - 用 `fs.existsSync` 自动判断

### 1.4 路径硬编码审查 + 修复

- [x] `frontend/src/components/Workspace/FileTreeTab.vue` — 已确认为 props 兜底，生产环境由 apiBase.ts 覆盖
