# 开发版升级用户版标准行为手册

> 章鱼博士出品 | 版本 1.0 | 2026-03-31

---

## 🎯 核心原则

**只升级，不污染！**

1. **开发版是沙盒** - `octowork-chat-dev/` 里随便折腾
2. **用户版是展厅** - `octowork-chat/` 必须干净整洁
3. **单向流动** - 功能验证OK后才从开发版复制到用户版
4. **端口统一** - 开发版和用户版都使用 **前端5888 + 后端1314**

---

## 📋 升级前检查清单

### ✅ 开发版功能验证
- [ ] 功能在开发版完整测试通过
- [ ] 无未完成的调试代码或临时日志
- [ ] 代码符合项目规范（ESLint检查通过）
- [ ] 数据库迁移脚本已准备（如有）

### 🧹 用户版环境检查
- [ ] 确认用户版目录位置：`/Users/jason/octowork-chat/`
- [ ] 备份当前用户版（重要！）
- [ ] 检查端口占用：1314和5888是否空闲

## 🔄 标准升级流程

### 阶段一：开发版准备 (5分钟)
```
1. 功能冻结 - 停止对该功能的开发
2. 代码整理 - 移除调试代码、console.log
3. 文档更新 - 同步API文档和用户指南
4. 打包准备 - 确认所有文件就绪
```

### 阶段二：用户版升级 (10分钟)
```
1. 备份用户版（必须！）
   cp -r /Users/jason/octowork-chat /Users/jason/octowork-chat-backup-$(date +%Y%m%d)

2. 复制核心代码
   # 前端代码
   rsync -av --exclude='node_modules' --exclude='.git' \
     /Users/jason/octowork-chat-dev/frontend/ \
     /Users/jason/octowork-chat/octowork-chat/frontend/
   
   # 后端代码  
   rsync -av --exclude='node_modules' --exclude='.git' \
     /Users/jason/octowork-chat-dev/backend/ \
     /Users/jason/octowork-chat/octowork-chat/backend/

3. 配置文件处理
   # 保留用户版特有的配置
   # 开发版config.json → 用户版config.json（手动合并，不覆盖关键配置）

4. 数据库处理（如有）
   # 运行迁移脚本
   # 备份原数据库
```

### 阶段三：验证测试 (5分钟)
```
1. 启动用户版服务
   cd /Users/jason/octowork-chat/octowork-chat/backend
   npm start
   
   cd /Users/jason/octowork-chat/octowork-chat/frontend
   npm run dev

2. 基础功能验证
   curl http://localhost:1314/api/bots        # 应返回Bot列表
   curl http://localhost:5888/                # 前端应正常响应

3. 新功能专项测试
   按照功能测试用例逐一验证
```

### 阶段四：清理归档 (2分钟)
```
1. 开发版归档
   git commit -m "功能X开发完成，已升级到用户版"
   
2. 用户版标记
   更新用户版版本号（package.json）

3. 文档同步
   更新用户版docs/相关文档
```

## 🚨 禁止行为 (红线！)

### ❌ 绝对不要做的
1. **不要** 直接将开发版整个目录复制到用户版
2. **不要** 覆盖用户版独有的配置文件（如生产环境配置）
3. **不要** 将测试数据或调试脚本复制到用户版
4. **不要** 在用户版运行npm install --save-dev（生产环境只装依赖）
5. **不要** 忘记备份直接操作

### ❌ 必须避免的
1. **避免** 在用户版保留console.log或调试代码
2. **避免** 用户版目录结构混乱（保持标准结构）
3. **避免** 端口冲突（1314和5888专用于用户版）
4. **避免** 数据库直接操作（必须通过迁移脚本）

## 📁 文件处理规范

### 可以复制的文件类型
- `.js`, `.vue`, `.html`, `.css` - 源代码文件
- `.json`（配置需手动合并）
- `.md`文档文件
- 图片资源（压缩优化后）

### 必须过滤的文件
- `node_modules/` - 依赖目录（用户版单独安装）
- `.git/` - 版本控制目录
- `*.log` - 日志文件
- `test*.js`, `*.spec.js` - 测试文件
- `temp/`, `tmp/`, `scratch/` - 临时目录

### 需要特殊处理的文件
```
config.json      # 手动合并，保留用户版生产配置
.env            # 不复制，用户版单独配置
package.json    # 对比依赖，只更新必要的版本
```

## 🔧 自动化升级脚本

创建 `/Users/jason/octowork-chat-dev/tools/upgrade-to-user-version.sh`：

```bash
#!/bin/bash
# 开发版升级用户版自动化脚本
# 使用前：1.备份 2.确认功能测试通过 3.检查配置

set -e

echo "🚀 开始升级流程..."

# 1. 备份用户版
BACKUP_DIR="/Users/jason/octowork-chat-backup-$(date +%Y%m%d_%H%M%S)"
echo "📦 备份用户版到: $BACKUP_DIR"
cp -r "/Users/jason/octowork-chat" "$BACKUP_DIR"

# 2. 复制前端代码（排除node_modules）
echo "🔄 复制前端代码..."
rsync -av --delete --exclude='node_modules' --exclude='.git' \
  "/Users/jason/octowork-chat-dev/frontend/" \
  "/Users/jason/octowork-chat/octowork-chat/frontend/"

# 3. 复制后端代码
echo "🔄 复制后端代码..."
rsync -av --delete --exclude='node_modules' --exclude='.git' \
  "/Users/jason/octowork-chat-dev/backend/" \
  "/Users/jason/octowork-chat/octowork-chat/backend/"

# 4. 特殊文件处理提醒
echo "📝 注意：以下文件需要手动处理："
echo "   - config.json (合并配置)"
echo "   - .env (环境变量)"
echo "   - package.json (依赖版本)"

echo "✅ 文件复制完成！请继续："
echo "   1. 手动处理特殊文件"
echo "   2. 启动服务验证"
echo "   3. 运行功能测试"
```

给脚本执行权限：`chmod +x tools/upgrade-to-user-version.sh`

## 🐛 常见问题与解决方案

### Q1: 升级后用户版服务启动失败
**可能原因**：
1. 端口被占用（1314/5888）
2. 依赖缺失（node_modules）
3. 配置文件错误

**解决方案**：
```bash
# 检查端口占用
lsof -i :1314
lsof -i :5888

# 重新安装依赖
cd /Users/jason/octowork-chat/octowork-chat/backend && npm ci
cd ../frontend && npm ci

# 检查配置文件
cat config.json | jq .  # 确保格式正确
```

### Q2: 数据库迁移失败
**解决方案**：
1. 恢复备份数据库
2. 检查迁移脚本权限
3. 分步执行迁移

### Q3: 新功能在用户版不生效
**排查步骤**：
1. 确认代码已正确复制（diff比较）
2. 检查浏览器缓存（硬刷新 Ctrl+Shift+R）
3. 查看服务日志（backend/server.log）
