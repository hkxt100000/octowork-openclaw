## 一、入口与挂载方式

群聊模块有 **两种入口**:

| 入口 | 路径 | WS 注册 | 说明 |
|------|------|---------|------|
| **App.vue 内嵌** | `/` 主页面，当 `currentType='group'` 时显示 | `registerWS: false` | 由 App.vue 的 useWebSocket 统一分发 |
| **GroupChatPage.vue 独立路由** | `/group/:groupId` | `registerWS: true` | composable 自注册 WS，自管理生命周期 |

| | `selectMention(mention)` | 选中 @mention: @all→'@全体成员', 其它→'@中文昵称' + 记录 mentionDisplayMap |

| | `handleGroupNotification(data)` | WS notification: ElNotification 右下角8s + 点击跳转对应群 |

### 3.3 消息缓存策略

```
sessionStorage key: 'group_messages_cache'
结构: { messages: Record<groupId, GroupMessage[]>, _timestamp: number }
过期: 30分钟 (CACHE_EXPIRE_MS = 30 * 60 * 1000)
写入时机: loadGroupMessagesLocal / loadMoreGroupMessages / handleGroupNewMessage
读取时机: composable 初始化时 loadMessagesFromCache()
```

### 3.4 40-45-5 消息保留策略

```
触发条件: 消息数 >= 45
动作: 删除前40条，仅保留最新5条
执行位置:
  1. loadGroupMessagesLocal() — 初始加载时
  2. handleGroupNewMessage() — WS 实时消息到达时
  3. allGroupMessages 缓存同步
```

### 3.5 WS 自注册机制

```
registerWS: true  → 初始化时 ws.on('group_message', handler) + ws.on('notification', handler)
registerWS: false → 不注册（由外部统一处理）

onUnmounted:
  - stopPolling()
  - ws.off('group_message', _wsGroupHandler)
  - ws.off('notification', _wsNotificationHandler)

使用场景:
  - App.vue: registerWS=false（useWebSocket 统一分发）
  - GroupChatPage: registerWS=true（独立页面自管理）
```

## 五、@mention 完整流程

```
用户输入 "@" 
  → handleGroupInputChange()
    → lastIndexOf('@') + 提取 searchText
    → 严格从 selectedGroup.members 构建 sourceBots（不回退到全局 bots）
    → members 为空时不显示建议列表（杜绝跨群泄漏）
    → 构建 @全体成员 快捷项（匹配 "全体成员" 或 "all"）
    → mentionSuggestions = [@all(if matched), ...filtered]

键盘导航:
  Shift+Enter → 换行（手动插入 \n）
  ArrowDown → selectedMentionIndex++
  ArrowUp   → selectedMentionIndex--
  Enter     → (有 mention) selectMention() / (无 mention) handleSendGroupMessage()
  Escape    → showMentionSuggestions = false

selectMention(mention):
  @all  → groupInputText = '@全体成员 ', mentionDisplayMap['全体成员'] = 'all'
  其它  → groupInputText = '@中文昵称 ', mentionDisplayMap[中文昵称] = botId

发送时:
  → resolveDisplayNamesToIds(text)  ← 将 @中文昵称/@全体成员 按长度降序替换回 @botId/@all
  → mentionDisplayMap 清空
  → POST /api/groups/:id/messages (content 中为 @botId / @all)

消息显示时:
  → highlightMentions(content)
  → @all   → <span class="mention mention-all">@全体成员</span> (橙色)
  → @botId → <span class="mention">@中文昵称</span> (蓝色)
```

## 六、图片上传流程

```
触发方式: + 按钮点击 / 拖拽到输入区 / Ctrl+V 粘贴

handleImageFile(file):
  → 类型检查: file.type.startsWith('image/')
  → 大小检查: file.size <= 10MB
  → pendingImage = file
  → pendingImageUrl = URL.createObjectURL(file)
  → 显示预览卡片(文件名+大小+缩略图)

发送时 (handleSendGroupMessage):
  → FormData POST /api/groups/:id/files → { file.url }
  → content += `![图片名](url)` (Markdown格式)
  → 清除 pendingImage + revoke ObjectURL

拖拽事件:
  handleDragOver → container.classList.add('drag-over') 显示蓝色覆盖层
  handleDragLeave → 移除覆盖层
  handleDrop → 提取图片文件 → handleImageFile

onUnmounted: URL.revokeObjectURL(pendingImageUrl) 防内存泄漏
```

## 十、与单聊模块的对比

| 维度 | 单聊 (Bot Chat) | 群聊 (Group Chat) |
|------|-----------------|-------------------|
| Composable | `useSingleChat.ts` | `useGroupChat.ts` |
| 消息组件 | `MessagesView.vue` | `GroupMessagesView.vue` |
| 输入组件 | `MessageInput.vue` | `GroupMessageInput.vue` |
| 头部组件 | `ChatHeader(type='bot')` | `ChatHeader(type='group')` |
| WS 事件 | `bot_message` | `group_message` + `notification` |
| @mention | 无 | 完整: 浮层+昵称映射+@all+键盘导航 |
| 图片上传 | 单聊图片上传 | 群聊图片上传 (multer, 10MB) |
| 消息缓存 | allBotMessages (内存) | allGroupMessages (内存+sessionStorage 30min) |
| 未读计数 | botUnreadCounts | groupUnreadCounts |
| 轮询兜底 | 30s, 仅页面可见+选中Bot | 30s, 仅页面可见+选中群 |
| 消息保留 | MAX_MESSAGES=200 | 40-45-5 策略 |
| 文件树面板 | WorkspacePanel | GroupWorkspacePanel |
