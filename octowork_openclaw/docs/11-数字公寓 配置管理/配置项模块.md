# 配置项模块

> OctoWork系统配置管理文档

---

## 🎯 功能概述

**核心功能**: 管理OctoWork聊天管理器的所有配置项，包括系统配置、环境变量、端口设置、功能开关等。

**业务价值**: 提供统一的配置管理机制，支持开发、测试、生产环境的差异化配置。

## ❓ 常见问题

### Q1: 如何修改后端端口？
**问题**: 默认端口6726被占用，需要修改端口号。

**解决方案**:
1. 修改 `backend/config.json` 中的 `port` 字段
2. 同时修改 `projects/bot-chat-manager/config.json` 中的 `port` 字段
3. 重启后端服务：`cd backend && npm start`

### Q2: 前端端口如何配置？
**问题**: 前端开发服务器端口需要修改。

**解决方案**:
1. 修改 `frontend/vite.config.ts` 中的 `server.port` 配置
2. 如果需要HTTPS，配置 `server.https` 选项
3. 重启前端服务：`cd frontend && npm run dev`

### Q3: 工作空间路径配置错误？
**问题**: 文件树空白，可能是workspacePath配置错误。

**解决方案**:
1. 检查 `backend/config.json` 中的 `workspacePath` 字段
2. 确保路径指向正确的OpenClaw工作空间
3. 使用绝对路径，避免相对路径问题

### Q4: 配置修改后服务不生效？
**问题**: 修改配置后需要重启服务。

**解决方案**:
1. 停止当前服务进程
2. 删除 `backend/logs/` 下的日志文件（可选）
3. 重新启动服务：执行 `START.sh` 脚本

## 🔧 修改指南

### 添加新配置项
1. **后端配置**:
   ```json
   // 在 backend/config.json 中添加
   {
     "newFeature": {
       "enabled": true,
       "options": {}
     }
   }
   ```

2. **前端配置**:
   ```javascript
   // 在 frontend/config.json 中添加对应配置
   ```

3. **类型定义**（如果使用TypeScript）:
   ```typescript
   // 在 frontend/src/types/config.ts 中添加类型定义
   export interface AppConfig {
     newFeature?: {
       enabled: boolean;
       options: Record<string, any>;
     };
   }
   ```

### 修改现有配置
1. **备份原配置**: 复制当前配置文件到备份位置
2. **编辑配置**: 使用文本编辑器修改配置项
3. **验证配置**: 调用 `/api/config/validate` 接口验证
4. **重启服务**: 配置生效需要重启相关服务

### 配置环境变量
```bash
# 开发环境
export NODE_ENV=development
export PORT=6726

# 生产环境  
export NODE_ENV=production
export PORT=80
```
