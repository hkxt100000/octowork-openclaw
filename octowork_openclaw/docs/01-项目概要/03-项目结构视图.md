## 📁 一级目录详解

### 1. `backend/` - 后端服务
**核心职责**: 提供API接口、WebSocket实时通信、数据库操作、消息处理等。

```
backend/
├── 📄 server.js                  # Express主服务文件 (核心，监听1314端口)
├── 📄 server.js.bak2/3          # 服务器备份文件 (历史版本)
├── 📄 server.log                 # 服务运行日志 (实时生成)
├── 📄 server.out                 # 服务输出日志
├── 📄 server_simple.js           # 简化版服务 (测试用)
├── 📄 server_dynamic_bot.js      # 动态Bot服务
├── 📄 server_legacy.js           # 遗留版本服务
├── 📄 package.json               # Node.js依赖配置
├── 📁 api/                       # API模块
│   └── 📄 openclaw.js           # OpenClaw网关API调用
├── 📁 services/                  # 业务服务层
│   ├── 📄 task-monitor.js       # 任务监控服务
│   ├── 📄 message-reconciliation-enhanced.js  # 消息协调服务 (增强版)
│   ├── 📄 messageSummaryService.js  # 消息总结服务
│   ├── 📄 fileWatcher.js        # 文件监控服务
│   └── 📄 timeout_checker.js    # 超时检查服务
├── 📁 db/                        # 数据库层
│   └── 📄 database.js           # SQLite数据库连接和操作
├── 📁 websocket/                 # WebSocket实时通信
│   └── 📄 manager.js            # WebSocket连接管理
├── 📁 utils/                     # 工具函数
│   ├── 📄 bot_env_injector.js   # Bot环境变量注入
│   ├── 📄 octo_notify.js        # 通知工具
│   ├── 📄 outbox-writer.js      # 发件箱写入工具
│   └── 📄 webhook.js            # Webhook工具
├── 📁 data/                      # 数据文件目录
│   ├── 📄 auto_healer_status.json  # 自动修复状态
│   ├── 📄 user.json             # 用户配置文件
│   └── 📁 offline_queue/        # 离线消息队列
├── 📁 avatars/                   # Bot头像存储
├── 📁 scripts/                   # 后端专用脚本
├── 📁 src/                       # 源代码 (可能包含TypeScript源文件)
├── 📁 tasks/                     # 定时任务定义
├── 📁 tests/                     # 后端测试文件
├── 📁 tools/                     # 后端专用工具
├── 📁 uploads/                   # 文件上传存储目录
└── 📁 node_modules/              # Node.js依赖包 (自动生成)
```

**快速定位**:
- 改API → `backend/api/`
- 改业务逻辑 → `backend/services/`
- 改数据库 → `backend/db/database.js`
- 查日志 → `backend/server.log`

### 2. `frontend/` - 前端应用
**核心职责**: 用户界面展示、交互逻辑、与后端API通信。

```
frontend/
├── 📄 package.json               # Vue项目依赖配置
├── 📄 vite.config.ts             # Vite构建配置 (TypeScript版本)
├── 📄 index.html                 # 主HTML入口文件
├── 📄 config.json                # 前端运行时配置
├── 📄 frontend.log               # 前端运行日志
├── 📄 dashboard-preview.html     # Dashboard预览页面
├── 📁 src/                       # 源代码目录 (核心)
│   ├── 📁 renderer/              # 渲染进程 (核心UI)
│   │   ├── 📄 App.vue            # 主应用组件 (5870行，持续拆分中)
│   │   ├── 📁 assets/            # 静态资源 (图片、字体等)
│   │   ├── 📁 components/        # 功能组件
│   │   ├── 📁 composables/       # 组合式函数 (Vue 3 Composition API)
│   │   ├── 📁 layouts/           # 布局组件
│   │   ├── 📁 services/          # 前端服务层 (API调用)
│   │   ├── 📁 styles/            # 样式文件
│   │   ├── 📁 types/             # TypeScript类型定义
│   │   ├── 📁 utils/             # 工具函数
│   │   └── 📁 views/             # 页面级视图组件
│   ├── 📁 components/            # 通用组件 (共享)
│   ├── 📁 main/                  # 主进程 (Electron相关，当前未使用)
│   └── 📁 utils/                 # 通用工具函数
├── 📁 public/                    # 静态资源目录 (直接复制到构建输出)
│   ├── 📄 manifest.json          # PWA配置
│   └── 📁 icons/                 # 应用图标
├── 📁 dist/                      # 构建输出目录 (自动生成)
├── 📁 avatars/                   # Bot头像缓存 (前端使用)
└── 📁 node_modules/              # 前端依赖包 (自动生成)
```

**快速定位**:
- 改页面UI → `frontend/src/renderer/views/`
- 改通用组件 → `frontend/src/renderer/components/`
- 改API调用 → `frontend/src/renderer/services/`
- 改样式 → `frontend/src/renderer/styles/`

### 6. `memory/` - 章鱼博士工作记忆
**核心职责**: Agent工作记忆和临时文件存储，**非项目代码**。

```
memory/

│   ├── 📁 daily_logs/            # 每日工作日志
│   ├── 📁 scratch/               # 临时文档和草稿 (重要!)

└── 📁 long_term/                 # 长期记忆 (知识库)
```

**重要规则**: 
- 所有临时文件必须放在 `memory/short_term/scratch/`
- 禁止在根目录或其他项目目录创建临时文件
- 这是章鱼博士的"个人公寓"，不是项目源代码

## 🚀 快速使用指南

### 新开发者入职
1. **了解结构** → 阅读本文档 (`docs/01-项目概要/项目结构视图.md`)
2. **启动项目** → 运行 `./START.sh` 或分别启动后端和前端
3. **修改后端** → 进入 `backend/` 目录，主要修改 `server.js` 和 `services/`
4. **修改前端** → 进入 `frontend/` 目录，主要修改 `src/renderer/`
5. **查看文档** → 进入 `docs/` 对应模块目录

### 快速定位文件
- **找API接口** → `backend/api/`
- **找前端页面** → `frontend/src/renderer/views/`
- **找组件** → `frontend/src/renderer/components/`
- **找文档** → `docs/XX-模块名-栏目开发/`
- **找配置** → `config.json` 或 `docs/11-数字公寓 配置管理/`

### 开发工作流
1. 修改代码前先确认文件位置 (参考本文档)
2. 测试功能是否正常
3. 如有必要，更新相关文档 (特别是结构变更时)
4. 提交代码，保持结构清晰
