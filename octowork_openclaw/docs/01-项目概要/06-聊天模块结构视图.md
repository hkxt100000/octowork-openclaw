## 🔌 API接口详解

### 1. 消息管理API
#### 获取消息历史
- **端点**: `GET /api/bots/:botId/messages`
- **参数**: `botId` (路径), `limit`, `before` (分页)
- **响应**: `{ success: true, messages: [...] }`
- **前端调用**: `useMessages.ts` → `loadMessages()`

#### 发送消息
- **端点**: `POST /api/bots/:botId/messages`
- **参数**: `botId` (路径), `content`, `mode` (聊天模式)
- **响应**: `{ success: true, messageId: '...' }`
- **前端调用**: `useMessages.ts` → `sendMessage()`

#### 删除消息历史
- **端点**: `DELETE /api/bots/:botId/messages`
- **参数**: `botId` (路径)
- **响应**: `{ success: true }`
- **前端调用**: `useMessages.ts` → `clearMessages()`

### 2. 文件操作API
#### 获取文件树
- **端点**: `GET /api/bots/:botId/file-tree`
- **参数**: `botId` (路径)
- **响应**: 文件树结构 (JSON)
- **用途**: 显示Bot工作空间文件结构

#### 读取文件内容
- **端点**: `GET /api/bots/:botId/files/:filePath`
- **参数**: `botId`, `filePath` (路径)
- **响应**: 文件内容
- **用途**: 在聊天中查看或编辑文件

### 3. 消息总结API
#### 总结单条消息
- **端点**: `POST /api/summarize/message`
- **参数**: `messageId`, `content`
- **响应**: 总结后的内容
- **状态**: ✅ 后端已实现，前端待集成

## 🔄 消息状态跟踪

### 消息生命周期
```
created → 创建消息对象，分配临时ID
sending → 发送到后端，显示"发送中"
sent → 后端确认收到，显示"已发送"
delivered → Bot确认收到，显示"已送达"
read → 用户查看消息，显示"已读"
failed → 发送失败，显示"发送失败"
```

### 状态指示器
- ⏳ **发送中**: 灰色圆圈动画
- ✅ **已发送**: 灰色对勾
- ✅✅ **已送达**: 蓝色双对勾
- 👁️ **已读**: 蓝色眼睛图标
- ❌ **失败**: 红色感叹号，点击重试

## 📊 性能考虑

### 当前优化
1. **乐观更新**: 发送消息时立即更新UI，不等待响应
2. **消息合并**: 短时间内多条消息可能合并显示
3. **图片优化**: 消息中的图片使用缩略图和懒加载
4. **WebSocket复用**: 共享WebSocket连接，减少连接数

3. **请求合并**: 多个API请求合并为批量请求
4. **组件懒加载**: 聊天组件按需加载

## 🔍 故障排查

### 常见问题1: 消息发送失败
**可能原因**:
1. WebSocket连接断开
2. 后端API服务异常
3. Bot服务不可用
4. 网络连接问题

**排查步骤**:
1. 检查WebSocket连接状态
2. 查看浏览器控制台网络请求
3. 检查后端服务日志 (`backend/server.log`)
4. 验证Bot服务状态

### 常见问题2: 消息不实时更新
**可能原因**:
1. WebSocket消息未正确处理
2. 消息协调服务异常
3. 前端状态更新逻辑错误

**排查步骤**:
1. 检查WebSocket消息接收处理
2. 查看消息协调服务日志
3. 使用Vue DevTools检查消息状态
4. 验证 `useMessages.ts` 中的消息更新逻辑

### 常见问题3: 文件树加载慢
**可能原因**:
1. Bot工作空间文件过多
2. 文件树API响应慢
3. 前端渲染性能问题

**排查步骤**:
1. 检查文件树API响应时间
2. 实现虚拟滚动优化
3. 添加加载状态和进度指示
4. 考虑分页加载文件树
