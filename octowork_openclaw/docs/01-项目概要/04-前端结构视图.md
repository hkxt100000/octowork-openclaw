## 🎨 前端应用概览

**技术栈**: Vue 3 + TypeScript + Vite + Pinia (规划中)
**核心架构**: 单页面应用 (SPA)，四列/三列布局动态切换
**开发服务器**: 端口5888 (开发版)
**构建工具**: Vite (TypeScript配置)

**当前状态**:
- **App.vue**: 5870行超大组件，持续拆分中
- **组件化**: 已有部分组件拆分，但核心逻辑仍在App.vue
- **状态管理**: 使用Composition API，尚未完全迁移到Pinia
- **路由**: 简单路由配置，页面级路由待完善

**核心布局**: 
- 第一列: 全局导航栏 (MainLayout)
- 第二列: 内容侧边栏 (ContentSidebar) 
- 第三列: 主内容区 (动态切换: 聊天/群聊/任务等)
- 第四列: 工作台/文件树 (WorkspacePanel)

## 🚨 常见问题与定位

### 问题1: 页面渲染异常
**排查步骤**:
1. 检查控制台错误 (F12开发者工具)
2. 检查`frontend.log`日志文件
3. 检查网络请求 (API调用是否成功)
4. 检查Vue DevTools中的组件状态

### 问题2: 消息发送失败
**排查步骤**:
1. 检查WebSocket连接状态
2. 检查后端API响应 (网络面板)
3. 检查`useMessages.ts`中的sendMessage函数
4. 检查消息验证逻辑

### 问题3: 样式异常
**排查步骤**:
1. 检查CSS变量是否正确定义
2. 检查scoped样式是否冲突
3. 检查主题切换逻辑
4. 使用浏览器开发者工具检查计算样式

### 问题4: TypeScript编译错误
**排查步骤**:
1. 检查类型定义文件 (`types/`目录)
2. 检查接口是否与后端API匹配
3. 检查泛型使用是否正确
4. 运行`tsc --noEmit`检查类型错误

## 📊 性能优化提示

### 当前已知性能问题
1. **App.vue过大**: 影响热更新速度，增加内存使用
2. **重复渲染**: 某些计算属性可能触发不必要的重新渲染
3. **图片加载**: Bot头像可能未使用懒加载

### 优化建议
1. **代码拆分**: 按路由懒加载页面组件
2. **组件懒加载**: 非首屏组件使用`defineAsyncComponent`
3. **图片优化**: 使用懒加载和WebP格式
4. **状态管理**: 迁移到Pinia，减少响应式依赖

## 📝 开发规范

### 组件设计原则
- **单一职责**: 每个组件只做一件事
- **可组合性**: 组件通过props和slots组合
- **可测试性**: 组件逻辑可独立测试
- **可维护性**: 清晰的props接口和文档

### 状态管理原则
- **响应式数据**: 使用ref/reactive管理状态
- **计算属性**: 使用computed派生状态
- **副作用**: 使用watch/watchEffect处理副作用
- **类型安全**: 所有状态都有TypeScript类型

### 样式编写原则
- **CSS变量**: 使用变量管理主题色和间距
- **作用域**: 优先使用scoped CSS
- **BEM命名**: 复杂组件考虑BEM命名规范
- **响应式**: 使用媒体查询适配不同屏幕
