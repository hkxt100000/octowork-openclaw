### 文件系统集成
**任务文件存储规则**:
1. **任务创建**: 在 `task_box/pending/` 创建任务文件
2. **状态变更**: 移动文件到对应状态目录
3. **任务完成**: 移动到 `task_box/completed/`，添加完成时间
4. **任务验收**: 移动到 `task_box/accepted/`，添加验收记录
5. **任务归档**: 定期将旧任务移动到归档目录

**文件监听机制**:
- 使用Chokidar监听 `task_box/` 目录变化
- 文件变化时自动更新前端状态
- 支持实时协作 (多用户同时编辑)

## 🎯 用户体验流程

### 1. 查看任务看板
```
用户进入任务模块 → 显示BoardPage → 
加载部门列表 → 选择部门 → 
加载该部门任务 → 显示KanbanView →
可切换时间线视图
```

### 2. 创建新任务
```
点击新建任务按钮 → 显示创建任务表单 →
填写任务信息 → 选择负责人 → 设置截止时间 →
保存 → 任务文件创建 → 显示在看板中 →
通知负责人
```

### 3. 更新任务状态
```
在看板中拖拽任务卡片 → 触发状态变更 →
更新任务文件位置 → 记录活动日志 →
通知相关人员 → 更新看板显示
```

### 4. 查看任务详情
```
点击任务卡片 → 打开TaskDetailModal →
显示任务完整信息 → 可编辑任务详情 →
查看评论和附件 → 跟踪活动记录
```
