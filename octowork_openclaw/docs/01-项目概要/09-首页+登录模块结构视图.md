## 🎯 用户体验流程

### 1. 登录流程
```
用户访问 → 显示Login.vue → 输入凭据 → 调用/auth/login → 
验证成功 → 保存token → 跳转到Dashboard → 加载用户数据
```

### 2. Dashboard访问流程
```
登录成功 → 加载DashboardView → 并行加载:
  1. 获取Bot列表 (useBots.fetchBots)
  2. 获取任务数据 (useTasks.fetchTasks)
  3. 建立WebSocket连接
  4. 加载用户偏好
→ 渲染各Dashboard组件 → 实时更新数据
```

### 3. 导航切换流程
```
用户点击导航项 → MainLayout触发view-change → 
App.vue更新currentView → 动态渲染对应内容:
  - dashboard: 显示DashboardView
  - chats: 显示聊天界面
  - groups: 显示群聊界面
  - tasks: 显示任务界面
→ 更新URL路由 (如果启用路由)
```

## 🔍 故障排查

### 常见问题1: 登录失败
**可能原因**:
1. 后端登录API未实现
2. 网络连接问题
3. 凭证验证失败

**排查步骤**:
1. 检查浏览器控制台网络请求
2. 验证后端 `/api/login` 端点是否存在
3. 检查 `auth.ts` 中的错误处理

### 常见问题2: Dashboard数据不显示
**可能原因**:
1. API调用失败
2. 数据格式不匹配
3. 组件渲染错误

**排查步骤**:
1. 检查 `useBots.ts` 和 `useTasks.ts` 的数据加载
2. 验证API返回数据结构
3. 使用Vue DevTools检查组件的props和数据

### 常见问题3: 布局异常
**可能原因**:
1. CSS样式冲突
2. 布局计算错误
3. 响应式断点问题

**排查步骤**:
1. 检查浏览器开发者工具中的样式计算
2. 验证 `App.vue` 中的布局条件逻辑
3. 测试不同屏幕尺寸下的表现
