## 🔄 分层架构说明

### 1️⃣ 路由层（Routes）

**职责**：定义API端点，不包含业务逻辑

```javascript
// src/routes/botRoutes.js
router.get('/messages/:botId', controller.getMessages)
router.post('/messages/:botId', controller.sendMessage)
```

**特点**：
- 只定义路由映射
- 调用对应的Controller方法
- 不包含任何业务逻辑

---

### 2️⃣ 控制器层（Controllers）

**职责**：处理HTTP请求，调用服务层完成业务逻辑

```javascript
// src/controllers/botController.js
class BotController {
  async sendMessage(req, res) {
    // 1. 参数验证
    // 2. 调用服务层
    // 3. 返回响应
  }
}
```

**特点**：
- 处理请求和响应
- 参数验证
- 错误处理
- 调用Service层

---

### 3️⃣ 服务层（Services）

**职责**：核心业务逻辑，可复用的服务

```javascript
// tasks/task_manager.js
class TaskManager {
  createTask(data) { /* 创建任务卡片 */ }
  updateTaskStatus(taskId, status) { /* 更新状态 */ }
}
```

**特点**：
- 独立的业务逻辑
- 可被多个Controller调用
- 不依赖HTTP请求

## 🎯 依赖注入模式

使用**依赖注入**，避免硬编码依赖：

```javascript
// server_modular.js

// 1. 准备依赖
const dependencies = {
  db,
  eventBus,
  taskManager,
  wsManager,
  // ...
}

// 2. 初始化控制器（注入依赖）
const controllers = initializeControllers(dependencies)

// 3. 注册路由
registerRoutes(app, controllers, services)
```

**优势**：
- ✅ 解耦：模块之间松耦合
- ✅ 可测试：方便单元测试（Mock依赖）
- ✅ 灵活：可替换实现

