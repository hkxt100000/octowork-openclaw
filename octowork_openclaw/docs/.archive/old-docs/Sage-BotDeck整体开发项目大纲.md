## 📋 项目概览

### 核心定位
**AI数字员工平台 - 按行业定制化的商业版Bot管理系统**

### 三大核心系统
1. **官网 + 后台管理系统**（统一Web项目）：用户注册 + 应用下载 + 管理后台
2. **botdeck-haoka**：号卡行业版桌面应用
3. **botdeck-kuajing**：跨境电商版桌面应用

### 技术架构
```
官网 + 后台管理（Vue 3前端 + Node.js后端）→ MySQL数据库
                    ↓ API
        botdeck-haoka.exe（Electron）
        botdeck-kuajing.exe（Electron）
                    ↓ 自动备份
                Gitee私有仓库
```

### 功能模块

#### 1.1 官网公开页面（无需登录）
**路由**：
- `/` - 首页（项目介绍、核心价值、行业案例）
- `/register` - 用户注册
- `/login` - 用户登录
- `/download` - 应用下载（需登录）

**核心流程 - 用户注册**：
1. 手机号 + 验证码
2. 选择行业（号卡/跨境）
3. ✅ **全部免费注册，无需选择套餐**
4. 提交后生成唯一账号编码（TH20260305001）
5. 显示编码，提示用户保存（用于首次登录应用）

**重要说明**：
- ❌ **不显示套餐选择**（个人版/公司版收费模块后期企微CRM再考虑）
- ❌ **不显示「成为合伙人」入口**（合伙人申请在应用内）

#### 1.2 后台管理系统（Jason专用）
**路由**：`/admin/*`

**用户管理**：
- 用户列表（分行业筛选）
- 用户详情（账号信息、设备绑定、Gitee仓库状态）
- 邀请推荐关系链
- 用户状态管理（启用/禁用）

**合伙人管理**：
- 分销合伙人申请审核（待审核/已通过/已拒绝）
- 行业合伙人申请审核
- 合伙人详情（推广业绩、团队结构）
- 佣金计算与结算
- 专属推广链接生成
- 提现申请审核

**内容管理**：
- 数字人市场管理（Agent上架审核、行业分类）
- 学习乐园管理（文章发布审核、行业分类）

**系统设置**：
- 行业管理（新增/编辑行业）
- 版本管理（应用版本发布）
- Gitee仓库管理

#### 1.3 合伙人后台（应用内Webview嵌入）
**路由**：`/partner/*`

- `/partner/dashboard` - 合伙人仪表盘
- `/partner/performance` - 推广业绩
- `/partner/commission` - 佣金明细
- `/partner/team` - 团队结构
- `/partner/withdraw` - 提现申请

### 技术栈
- 前端：Vue 3 + Element Plus + Vite
- 后端：Node.js + Express（或 Koa）
- 数据库：MySQL
- 认证：JWT
- 部署：服务器（Nginx反向代理）

## 🎯 系统二：botdeck-haoka（号卡版）

### 功能模块

#### 2.1 首次登录流程（关键）
1. 用户打开应用，弹窗提示「首次使用，请输入唯一编码」
2. 输入唯一编码（TH20260305001）+ 手机号 + 验证码
3. 调用后台API：
   - 验证账号信息
   - 验证行业匹配（账号行业 == 号卡）
   - **创建Gitee私有仓库**（仓库名 = 唯一编码）
   - 生成SSH密钥对
   - 将公钥添加到Gitee Deploy Key
   - 返回私钥给应用，本地保存
4. 初始化本地数据库（SQLite）
5. 配置定时任务（每天3点自动推送到Gitee）
6. 进入主界面

#### 2.2 后续登录流程
1. 输入手机号 + 验证码（或账号密码）
2. 调用后台API验证
3. 检测行业匹配
4. 返回用户信息 + `partner_info`（如果是合伙人）
5. 进入主界面

#### 2.3 核心功能（统一模块）
- 🏠 首页
- 💬 聊天（1对1）
- 👥 群聊
- ✅ 任务
- 📊 数据
- ⚙️ 设置
  - 账号信息（显示唯一编码）
  - 设备管理
  - **🔥 成为合伙人**（申请入口）
    - [申请分销合伙人]
    - [申请行业合伙人]

#### 2.4 云端功能（从后台API获取）
- 🤖 数字人市场（只展示号卡行业Agent）
- 📚 学习乐园（只展示号卡行业文章）

#### 2.5 行业独有模块
- 📱 号卡管理
- 💰 佣金计算
- 👥 代理招募

#### 2.6 合伙人管理（动态显示）
- 登录时检测 `partner_info` 是否存在
- 存在 → 左侧菜单自动增加「👔 合伙人管理」
- 点击 → Webview嵌入 `/partner/dashboard?token=xxx`
- 展示推广业绩、佣金、团队、提现等

### 技术栈
- 框架：Electron + Vue 3
- 复用代码：基于 `bot-chat-manager` 90%代码
- 本地数据库：SQLite
- 远程同步：Gitee私有仓库（每天3点自动推送）

## 🔑 核心技术方案

### 1. 用户注册与身份升级流程
```
步骤1：官网注册
→ 手机号 + 验证码 + 选择行业
→ 生成唯一编码（TH20260305001）
→ 用户状态：普通用户

步骤2：首次登录应用
→ 输入唯一编码 + 手机号 + 验证码
→ 后台创建Gitee私有仓库
→ 配置SSH密钥 + 定时推送

步骤3：申请成为合伙人（应用内）
→ 设置 → 成为合伙人 → 选择类型
→ 提交申请 → 等待审核

步骤4：审核通过后自动升级
→ partner_type字段更新
→ 下次登录显示「合伙人管理」菜单
```

### 2. 行业隔离机制
- 用户注册时绑定 `industry` 字段（haoka/kuajing）
- 登录时验证：`账号.industry == 应用.industry`
- 不匹配 → 拒绝登录并提示"您注册的是XX版，请下载XX版应用"

### 3. 合伙人身份识别
- users表字段：`partner_type`（null/distributor/industry）
- 登录API返回：`partner_info`（如果是合伙人）
- 应用检测 `partner_info` → 动态显示「合伙人管理」菜单

### 4. Gitee仓库管理
**创建流程**：
1. 用户首次登录应用，输入唯一编码
2. 后台调用Gitee API创建私有仓库（仓库名 = 唯一编码）
3. 生成SSH密钥对
4. 公钥添加到Gitee仓库Deploy Key（只写权限）
5. 私钥返回给应用，保存到本地 `~/.ssh/`
6. 更新users表：`repo_initialized = true`

**自动推送**：
- 应用注册定时任务（每天凌晨3点）
- 推送内容：Bot配置文件、IDENTITY、MEMORY、TOOLS、文档
- 不推送：data.db 数据库文件

### 5. 数字人市场与学习乐园过滤
- 数据库表增加 `industry` 字段
- API根据用户行业过滤：`WHERE industry = 用户.industry`

### 6. 代码复用策略
- 每个行业版 = 独立Git仓库 + 完整代码副本
- 公共模块：聊天/任务/数据/设置（从 bot-chat-manager 复用）
- 行业模块：各自独立开发

### 7. 应用更新机制
- 后台维护版本号 + 下载地址
- 应用启动时调用 `GET /api/version/check?app=haoka`
- 有新版本 → 后台下载 → 提示重启

## 📌 重要决策记录

### 决策1：独立仓库策略（2026-03-05）
- 每个行业版 = 独立仓库 + 完整代码
- 不抽取公共npm包
- 原因：专人维护专业仓库，互不影响

### 决策2：行业隔离规则（2026-03-05）
- 用户不能切换行业
- 一个手机号可注册多个行业账号
- 登录时严格验证行业匹配

### 决策3：合伙人身份升级模式（2026-03-05）
- 合伙人 = 用户 + 额外身份
- 先注册用户 → 后申请合伙人 → 审核通过后升级
- 官网不显示合伙人注册入口，只在应用内申请

### 决策4：免费注册策略（2026-03-05）
- 全部免费注册，无需选择套餐
- 个人版/公司版收费模块暂不开发
- 后期企微CRM系统再考虑付费

### 决策5：Gitee私有仓库管理（2026-03-05）
- 首次登录时创建Gitee私有仓库
- 仓库名 = 唯一账号编码
- 每天3点自动推送备份
- 换设备时从Gitee恢复数据

### 决策6：合伙人管理嵌入方式（2026-03-05）
- 前期：Webview嵌入
- 后期：优化为原生模块
