## 📝 开发笔记

### 重要设计决策

1. **task_box作为单一数据源**
   - 部门task_box是唯一真实数据源
   - Agent个人视图只读，避免双向同步冲突
   - 所有状态变更只发生在部门task_box

2. **事件总线解耦三系统**
   - 群聊、画板、单聊通过事件总线通信
   - 任意系统异常不影响其他系统
   - 新增功能只需订阅事件

3. **确认机制防止误触发**
   - 识别关键词后先显示预览
   - 必须点击确认才创建任务
   - 降低随便聊天误创建任务的概率

4. **离线通知持久化**
   - 用户不在线时通知存入文件
   - 避免服务重启后通知丢失
   - 上线时自动推送pending通知

5. **环境变量注入令牌**
   - 不在文件系统存储敏感信息
   - OctoWork启动Bot时注入环境变量
   - 提高安全性

### 性能优化要点

1. **文件监控优化**
   - chokidar使用`ignoreInitial: true`
   - 限制监控深度`depth: 2`
   - 避免监控大量无关文件

2. **WebSocket推送优化**
   - 只推送给订阅该部门的客户端
   - 使用debounce防止短时间多次推送
   - 批量更新合并为一次推送

3. **任务卡渲染优化**
   - 虚拟滚动（仅渲染可见卡片）
   - 卡片图片懒加载
   - 拖拽时使用CSS transform（GPU加速）

4. **本地缓存策略**
   - task_box内容缓存到内存
   - 定时（如每小时）同步到AI Drive
   - 减少远程文件系统IO
