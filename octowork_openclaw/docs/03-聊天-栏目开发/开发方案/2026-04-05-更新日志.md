## 一、电池系统全栈实装

### 后端 (`backend/src/controllers/botController.js`)

| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /api/bots/:botId/battery` | `getBatteryStatus` | 查询电池状态（token 使用量、模型信息） |
| `POST /api/bots/:botId/charge` | `triggerCharge` | 触发充电（摘要→清空→写回） |

**getBatteryStatus 流程**：
1. 优先调用 OpenClaw `sessions` 命令获取精确 token 用量
2. 失败时回退到本地消息数据库估算（字符数 x 1.3）
3. 返回 percent / level / token_used / token_limit / model / openclaw_available

**triggerCharge 8 步流程**：
1. 记录充电前 token 用量
2. 发送摘要 prompt（Bot 生成结构化摘要）
3. 提取 `---充电摘要开始---` / `---充电摘要结束---` 中的摘要
4. 持久化到 `memory/charge_summary_{timestamp}.md`
5. 清空 OpenClaw session（CLI → reset-flag → fallback）
6. 重置本地 session manager
7. 将摘要写回新 session
8. 记录充电日志 → WS 广播 `battery_update`

### 前端

| 文件 | 变更 |

### Bug 修复

| 问题 | 原因 | 修复 |
|------|------|------|
| `getSessionStatus is not a function` (500) | `server.js` 内联了简化版 `OpenClawClient`（仅 sendMessage），未包含 getSessionStatus | 删除内联类，改为 `require('./api/openclaw')` 导入完整版 |
| `sessions` 命令替代 `session-status` | OpenClaw CLI 没有 `session-status` 子命令 | 改用 `openclaw sessions` + JSON 解析 |

## 二、群聊功能增强

### 2.1 群聊列表 — 最新消息 + 未读红标

| 文件 | 变更 |
|------|------|
| `useGroupChat.ts` → `loadGroupsLocal` | `groupLastMessages` 始终从 API 更新（移除 `!groupLastMessages[g.id]` 条件） |
| `useGroupChat.ts` → `handleSendGroupMessage` | 发送后立即更新 `groupLastMessages`，不等 WS 推送 |

**已验证正常的机制**：
- `MainLayout.vue` 导航栏红标：绑定 `totalUnreadGroupCount`（computed 汇总所有群未读）
- `ContentSidebar.vue` 列表红标：绑定 `groupUnreadCounts[group.id]`（每群独立）
- `sortedGroups` 计算属性：有未读的群排前面，然后按最后消息时间倒序

### 2.2 群聊聊天区域 — 自动滚底

| 文件 | 变更 |
|------|------|
| `useGroupChat.ts` → `selectGroup` | 改为 `await loadGroupMessagesLocal` + `await nextTick()` |
| `GroupChatView.vue` | 新增 `watch(selectedGroup.id)` + 100ms 延迟强制 `scrollToBottom()` |
| `GroupMessagesView.vue` | 初始加载 watcher 增加 150ms `setTimeout` 确保异步内容渲染后滚底 |

### 2.3 40-45-5 消息删除策略

> 规则：前端消息数 ≥45 条时，删除前 40 条，仅保留最新 5 条

| 触发点 | 位置 |
|--------|------|
| `loadGroupMessagesLocal` | API 返回 ≥45 条 → `msgs.slice(-5)` |
| `handleGroupNewMessage` | WS 实时消息推入后 ≥45 条 → `groupMessages.slice(-5)` |
| `allGroupMessages` 缓存 | 同步裁剪，防 sessionStorage 膨胀 |

### 2.4 实时聊天记录文档

**后端** (`groupController.js` → `appendToGroupChatHistory`)：

| 特性 | 说明 |
|------|------|
| 自动创建 | 首条消息触发时自动创建 `{YYYY-MM-DD}.md` 文件 |
| 倒序排列 | 新消息插入在头部 `---` 之后，最新记录始终在文档最前 |
| 用户消息格式 | `### 👤 名称 \`HH:MM:SS\`` + blockquote 引用块 |
| Bot消息格式 | `### 🤖 名称 \`HH:MM:SS\`` + 正文 |
| 头部信息 | 表格式：日期 / 群组 / 消息数 / 排序方式 |

**前端** (`GroupWorkspacePanel.vue`)：

预览区 Markdown 样式全面优化 — h1 蓝色底线、blockquote 绿色引用（区分用户消息）、表格、代码高亮、暗色模式适配。
