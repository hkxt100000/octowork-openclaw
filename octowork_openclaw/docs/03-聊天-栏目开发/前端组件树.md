#### 消息轮询兜底机制

- `startPolling()` / `stopPolling()`: 每 30s (`POLLING_INTERVAL_MS`) 从 API 拉取最新消息
- 未读数轮询: 每 60s (`UNREAD_POLLING_MS`) 刷新所有Bot未读数
- 切换 Bot 时自动重启轮询 (watch selectedBot)
- 仅在页面可见 (`document.visibilityState === 'visible'`) 且有选中 Bot 时执行
- 用途: 即使 WebSocket 推送失败，消息也能 100% 到达前端

## 六、单聊消息流程

```
用户输入 → MessageInput.vue @send
→ useSingleChat.handleSend()
  → 有文件? uploadAndSendFile() : POST /api/messages/{botId} {content, style, userId, model}
  → 本地即时追加 userMessage (乐观更新)
  → 更新 allBotMessages 缓存 + saveMessagesToCache()
  → scrollToBottom()
  → 后台 loadMessagesLocal(botId) 刷新真实数据
  → 后台 loadRecentSessions() 刷新最近会话

Bot 回复到达 (两条路径):
  路径A: WS → useWebSocket handler
    → 收到 {type:'bot_message'} → messages.push(botMessage) + scrollToBottom()
    → 更新 allBotMessages 缓存 + botUnreadCounts
  路径B: 轮询兜底 (30s interval)
    → loadMessagesLocal(botId) → 拉取到新消息 → 自动更新
```

## 七、群聊消息流程

```
用户输入 → GroupMessageInput.vue @send-message
→ useGroupChat.handleSendGroupMessage(image)
  → 提取 @mentions → POST /api/groups/{groupId}/messages {content, mentions, style, userId}
  → 乐观更新 groupMessages + scrollToBottom()
  → 后端 _triggerMentionedBots → Bot异步回复
  
Bot 群聊回复:
  → POST /api/bot/send-to-group → saveGroupMessage → WS 广播 {type:'group_message'}
  → 前端 useWebSocket handler → groupMessages.push() / groupUnreadCounts++
```

## 十、初始化流程 (App.vue onMounted)

```
1. authService.getCurrentUser() → 验证登录
2. 注册闲置超时计时器 (8分钟, IDLE_TIMEOUT_MS)
3. loadMessagesFromCache() → 从 sessionStorage 恢复消息
4. loadBotsLocal() → 加载Bot列表 + ai-directory.json
5. setupCacheWatcher() → 注册消息缓存观察器
6. loadGroupsLocal() → 加载群组 + lastMessage
7. loadTasksLocal() → 加载任务看板
8. loadAllBotMessages() → 加载所有Bot消息缓存
9. loadRecentSessions() → 加载最近会话
10. connectWebSocket() → 建立WS连接
11. 注册 visibilitychange / online 事件监听
12. startTasksAutoRefresh(30s) → 任务定时刷新
13. startTeamsAutoRefresh(5min) → 团队定时刷新
14. startMessagesAutoRefresh(10s) → 消息定时刷新
15. startGroupPolling() → 群消息轮询
```

注意: 各步骤独立 try-catch，后端不可用时不阻塞整个初始化。
