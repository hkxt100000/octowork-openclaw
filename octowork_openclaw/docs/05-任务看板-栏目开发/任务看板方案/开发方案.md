## 一、开发策略：两阶段交付

### 为什么先做静态页面？

1. **降低返工风险**：UI/UX 先对齐，避免接完数据后发现布局要大改
2. **快速可视化**：用硬编码 Mock 数据，1-2 小时就能看到完整效果
3. **验收标准明确**：三个页面各长什么样，一目了然
4. **前后端可并行**：静态页面验收的同时，后端 API 可以同步开发

```
阶段一: 静态页面 (本轮)           阶段二: 完整实现 (验收后)
┌────────────────────┐          ┌────────────────────┐
│  硬编码 Mock 数据    │          │  真实 API 数据       │
│  三个页面完整 UI     │  验收通过  │  WebSocket 实时推送   │
│  交互效果 + 动画     │ ───────→ │  催促/跳转群聊功能    │
│  响应式布局          │          │  Guardian 实时状态    │
│  路由跳转正常        │          │  pipeline_state 监听  │
└────────────────────┘          └────────────────────┘
   预计 1.5-2 小时                    预计 4-5 小时
```
