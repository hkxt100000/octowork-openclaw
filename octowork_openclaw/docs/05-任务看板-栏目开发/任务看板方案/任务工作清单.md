## 一、总体架构回顾

```
pipeline_state.json (唯一真相源，每个项目一份)
        │
        ├── Bot 执行完任务后写入
        ├── Guardian 每 5 分钟写心跳
        └── QC 质检后更新状态
        │
        ▼
TaskBoxWatcher 监听文件变化 (chokidar)
        │
        ▼
WS 广播 { type: 'pipeline_update', ... }
        │
        ▼
前端看板实时刷新 (usePipeline.ts)

看板是【只读展示层】，不写 pipeline_state.json
催促按钮走旁路：看板 → API → 群聊 → Bot → pipeline → 看板
```

## 六、风险与阻塞项

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|----------|
| 工作区中暂无 `pipeline_state.json` 真实文件 | 确定 | 中 | API 实现空数据兼容：无文件 → 返回空列表，不报 500 |
| 群聊 Bot 触发修复未完成 | 确定 | 高 | Phase 4 催促功能依赖此修复；Phase 1-3 不受影响 |
| `pipeline_state.json` 频繁写入导致 WS 风暴 | 低 | 低 | chokidar `awaitWriteFinish` + debounce(300ms) |
| 旧组件被其他模块引用 | 低 | 中 | 删除前 `grep -rn` 全局搜索确认 |
| 跨部门 pipeline 模板尚未定义 | 确定 | 低 | Phase 1-4 只用 TokVideoGroup 真实数据验证 |
