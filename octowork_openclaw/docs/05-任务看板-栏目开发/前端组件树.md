## 一、入口方式

任务看板有 **两种入口**，共享同一套组件但导航方式不同：

| 入口 | 组件 | 导航 | 说明 |
|------|------|------|------|
| **App.vue 内嵌** | `TaskBoardContainer.vue` | 内部状态驱动 `currentPage` | 侧边栏切换到"任务看板"时渲染，不走 vue-router |
| **vue-router 独立路由** | 各子页面直接挂载 | 浏览器 URL 路由 | `/board` `/board/:deptId` `/board/:deptId/:projectId` |

### router.ts 路由配置（实际代码）

```typescript
{ path: '/board',                    name: 'Board',          component: () => import('./components/TaskBoard/BoardPage.vue') }
{ path: '/board/:deptId',            name: 'ProjectList',    component: () => import('./components/TaskBoard/ProjectListView.vue') }
{ path: '/board/:deptId/:projectId', name: 'PipelineDetail', component: () => import('./components/TaskBoard/PipelineView.vue') }
```

## 五、步骤状态机

```
blocked  ──(前置步骤 passed)──→  ready
ready    ──(Bot 开始执行)──→      in_progress
in_progress ──(Bot 完成)──→      completed
completed ──(QC 通过)──→         passed
completed ──(QC 打回)──→         rejected
rejected  ──(Bot 重做完成)──→    completed  (循环)
任何状态  ──(失败)──→            failed     (需人工介入)
```

### 状态色映射 (实际代码)

| 状态 | 颜色 | 标签 | emoji |
|------|------|------|-------|
| blocked | `#636366` (灰30%) | 等待中 | ⬜ |
| ready | `#007AFF` (蓝闪) | 就绪 | 🔵 |
| in_progress | `#007AFF` (蓝) | 执行中 | 🔄 |
| completed | `#FF9500` (橙) | 待质检 | 🟠 |
| passed | `#34C759` (绿) | 已通过 | ✅ |
| rejected | `#FF3B30` (红) | 已打回 | 🔴 |
| failed | `#FF3B30` (红闪) | 失败 | ❌ |

## 七、与群聊模块的联动

1. **群聊任务检测**: `sendGroupMessage` → `TaskDetector.detectTaskIntent()` → 生成 `task_preview` WS 推送 → 前端弹出 `TaskDetailDialog`
2. **任务确认**: 用户点击确认 → `POST /api/tasks/confirm` → 创建 task_box 文件 → WS 广播 `task_created`
3. **看板催促跳群聊**: PipelineView `handleGotoChat()` → `window.dispatchEvent('octowork-navigate')` → App.vue 监听跳转到群聊
4. **催促发消息**: `POST .../nudge` → eventBus `pipeline.step.started` → 群聊模块接收并在群聊发送催促消息
