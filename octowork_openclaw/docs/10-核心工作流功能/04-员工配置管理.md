## 📋 AI员工档案系统是什么？

**AI员工配置文件** (`ai-directory.json`) 是你的AI公司**完整人事档案**，它记录了：

- 👤 **员工基本信息** - 姓名、ID、头像、状态
- 🏢 **组织架构** - 部门、团队、汇报关系  
- 🎯 **角色职责** - 岗位、职责描述、技能标签
- 🧠 **能力配置** - 使用的AI模型、工作空间
- 📊 **活跃状态** - 最后活跃时间、工作状态

**一句话解释**：这是你AI公司的"员工花名册+组织架构图+岗位说明书"。

```json
{
  "bot_id": "OctoWork_Steward",           // 唯一ID，不可重复！

  "status": "active",                     // 状态: active/inactive/disabled
  "employee_id": "OCTOWORK-CORE-001"      // 工号，部门+序号
}
```

### 2. 组织架构信息
```json
{
  "department": "OctoWork",               // 所属部门
  "team": "octowork-core",                // 团队/小组
```

### 3. 能力与工作配置
```json
{
```

## 🚀 如何管理员工档案

### 场景1：添加新员工
```json
// 在 employees 对象中添加新条目
{
  "新员工_bot_id": {
    "bot_id": "new_employee",
    "chinese_name": "新员工",
    "english_name": "New Employee", 
    "display_name": "新员工 (测试岗位)",
    "department": "测试部门",
    "team": "test-team",
    "role": "测试工程师",
    "model": "deepseek/deepseek-chat",
    "workspace": "/departments/测试部门/agents/new_employee",
    "status": "active",
    "avatar": "🆕",
    "is_leader": false,

  }

```

**添加后必须**：
1. 更新 `total_employees` 计数
2. 如果新部门，更新 `total_departments`
3. 更新 `last_updated` 时间戳

```json
{
  "update_mechanism": {

      "departments/OctoWork/config/team_config.json",
      "departments/OctoVideo/agents/team_config.json"

    "scan_interval": 3600,                   // 每小时扫描一次

  }

```

### 手动更新流程

## 🧪 配置验证工具

### 验证脚本 validate_ai_directory.py
```python
#!/usr/bin/env python3
"""
验证ai-directory.json配置的完整性
"""
import json
import os
import sys

def validate_config(config_path):
    with open(config_path, 'r', encoding='utf-8') as f:
        config = json.load(f)
    
    errors = []
    warnings = []
    
    # 检查必需字段
    required_fields = ['version', 'total_employees', 'employees']
    for field in required_fields:
        if field not in config:
            errors.append(f"缺少必需字段: {field}")
    
    # 验证员工数量
    if 'employees' in config:
        actual_count = len(config['employees'])
        declared_count = config.get('total_employees', 0)
        
        if actual_count != declared_count:
            errors.append(f"员工数量不匹配: 声明{declared_count}个，实际{actual_count}个")
    
    # 检查每个员工
    for emp_id, employee in config.get('employees', {}).items():
        # 检查必需字段
        emp_required = ['bot_id', 'chinese_name', 'department', 'model']
        for field in emp_required:
            if field not in employee:
                warnings.append(f"员工 {emp_id} 缺少字段: {field}")
    
    return errors, warnings

if __name__ == "__main__":
    config_path = os.path.expanduser("~/octowork/config/ai-directory.json")
    errors, warnings = validate_config(config_path)
    
    if errors:
        print("❌ 发现错误:")
        for error in errors:
            print(f"  - {error}")
        sys.exit(1)
    
    if warnings:
        print("⚠️  发现警告:")
        for warning in warnings:
            print(f"  - {warning}")
    else:
        print("✅ 配置文件验证通过")
```

### 使用验证工具
```bash
# 运行验证
python3 validate_ai_directory.py

# 输出示例：
# ✅ 配置文件验证通过
# 或
# ❌ 发现错误:
#   - 员工数量不匹配: 声明17个，实际16个
```

## 🆘 常见问题解决

### 问题1：员工ID冲突
**现象**：添加新员工时报错"bot_id already exists"

**解决**：
1. 检查整个文件搜索该`bot_id`
2. 确保`bot_id`全局唯一
3. 建议命名规则：`部门_功能_序号`，如`tech_web_001`

### 问题2：部门统计错误
**现象**：`total_departments`与实际部门数不符

**解决**：
```bash
# 自动计算部门数
cat ai-directory.json | grep '"department":' | cut -d'"' -f4 | sort -u | wc -l

# 更新配置文件
departments_count=$(cat ai-directory.json | grep '"department":' | cut -d'"' -f4 | sort -u | wc -l)
sed -i '' "s/\"total_departments\":.*/\"total_departments\": $departments_count,/" ai-directory.json
```

### 问题3：路径不存在
**现象**：AI启动失败，报错"workspace not found"

**解决**：
1. 检查`workspace`字段的路径是否存在
2. 路径应该是相对`octowork`根目录的
3. 确保目录结构正确：
   ```bash
   # 示例路径
   ls -la ~/octowork/departments/OctoWork/agents/OctoWork_Steward/
   
   # 如果不存在，创建目录
   mkdir -p ~/octowork/departments/OctoWork/agents/OctoWork_Steward/
   ```

### 问题4：模型不可用
**现象**：AI无法响应，报错"model not available"

**解决**：
1. 检查`model`字段值是否正确
2. 确认OpenClaw配置中定义了该模型
3. 检查API密钥和网络连接
