  - Step 3: **程序主动删除** OpenClaw 远端会话（`deleteSession`，不依赖 AI）
  - Step 4: 生成全新版本化 sessionId（`user_xxx_bot_xxx_v{timestamp}`）+ 重置本地管理器
  - Step 5: 清空本地消息数据库
  - Step 6: WebSocket 广播充电完成 → **立即返回前端（电量=100%）**
  - ──── 以下步骤异步执行，不阻塞前端 ────
  - Step 7: 异步将摘要发送到全新 session（"这是你的上次总结，请继续"）
  - Step 8: 异步查询写回后 token + WebSocket 更新精确电量

## 三、技术决策记录

### 决策 1: Token 估算 fallback

当 `openclaw session-status` 不可用时，不显示"假满电"，而是：
1. 从 `messages` 表统计该 session 所有消息总字符数
2. 按 `字符数 x 1.3`（中英混合系数）粗算 token
3. 前端标注 "估算值（OpenClaw 不可用）"

### 决策 2: Session 清空策略 (V2.1 更新)

**核心原则：程序主导，AI 不参与清空**

1. `openclaw clear-session --agent <botId> --session-id <id>` CLI 命令
2. `openclaw sessions --agent <botId> --delete --session-id <id>` 删除命令
3. **始终生成全新版本化 sessionId** `user_{userId}_bot_{botId}_v{timestamp}`
   - 即使远端删除成功，也用全新 sessionId，彻底隔离旧上下文
   - `getUserBotSession()` 优先返回最新版本化 session

### 决策 3: 充电摘要持久化

每次充电的摘要保存到两个位置：
1. **memory/** 目录：Markdown 格式的可读摘要（含时间、session、token 信息）
2. **chat_history/** 目录：JSON 格式的充电日志（含完整元数据）

### 决策 4: 电池按钮交互

- 点击电池按钮 = 打开面板 + 自动查询最新状态
- 手动充电 = 确认对话框 → AI生成摘要 → 程序删除会话 → 全新session写回摘要 → 通知
- 自动充电 = 电量 <= 20% 时自动触发（前端 `checkAutoCharge`），不弹确认框

### 决策 5: 充电超时策略 (V2.2 更新)

- 前端 axios 超时设为 120 秒
- **V2.2 改进**：后端在 Step 3（删除会话）完成后立即返回，不等摘要写回
  - 摘要写回异步执行，即使花几十秒也不影响前端
  - 超时风险大幅降低（只有 Step 2 AI 生成摘要可能耗时较长）

### 决策 6: SessionId 版本化策略 (V2.1 新增)

**问题**：旧流程中 `getUserBotSession()` 始终返回固定格式 `user_xxx_bot_xxx`，充电后的新 session 无法被后续对话使用。

**方案**：
- 充电时生成 `user_{userId}_bot_{botId}_v{timestamp}` 格式的新 sessionId
- `getUserBotSession()` 遍历内存 Map，优先返回最新的版本化 session
- 无版本化 session 时回退到标准格式（兼容未充电过的 Bot）
- 进程重启后内存 Map 清空 → 自动回退到标准格式 → 开启新的标准 session（可接受）
